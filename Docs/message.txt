Project overview

Build a full-stack web application called SFPCL Member Credit Portal for Sahyadri Farmers Producer Company Limited (SFPCL). The application digitises the company's entire member loan lifecycle — from initial application by a farmer or FPC, through credit assessment, sanction committee approval, documentation, disbursement, and through to monitoring, repayment, and loan closure.

The SOP governing this application is SOP_SFPCL_LOANDISBURSEMENT v1.0, dated August 2025, under Section 378ZK of the Companies Act 2013.

Tech stack (recommended)
Frontend: React 18 + TypeScript + Tailwind CSS + React Router v6
Backend: Node.js + Express (or Next.js App Router for full-stack)
Database: PostgreSQL with Prisma ORM
Auth: Role-based JWT auth (5 distinct roles)
File storage: AWS S3 or Cloudflare R2 (for document uploads)
PDF generation: Puppeteer or React-PDF (for term sheets, NOCs)
Email/SMS: Nodemailer + Twilio (for notifications)
ERP integration: SAP BAPI or REST connector (loan posting, customer code creation)
User roles and permissions auth critical

The application must enforce these five roles. Every route and every action is gated by role.

1. Borrower (Farmer / FPC)
Submit loan application form with KYC documents
View own loan status, disbursement advice, interest invoices
Receive SMS/email notifications at each stage
View repayment schedule and outstanding balance
Cannot see any other borrower's data
2. Credit Assessment Team (Credit Manager + Deputy Manager Finance)
View all incoming loan applications
Prepare and submit Loan Appraisal Notes (Annexure B)
Calculate loan limits (both methods), flag eligibility issues
Issue rejection notes (Annexure L) to borrowers
Maintain Loan Request Register (auto-generated, sequential IDs: LO00000001…)
Update DPD buckets, send payment reminders, generate interest invoices
Cannot approve loans or disburse funds
3. Compliance Team (Company Secretary + Compliance Member)
Generate and manage all legal documents: PoA, Term Sheet, Loan Agreement, SH-4, Tri-Party Agreement, Bank Verification Letter, Checklist
Mark documents as stamped / notarised
Sign off on the checklist (CS signature = "documents verified & attached")
Manage KYC re-verification cadence (trigger re-KYC every 2 years)
Issue NOC and return SH-4 + blank cheque on loan closure
Manage grievance register
Cannot approve or disburse
4. Sanction Committee (CFO + Executive Directors)
View Loan Appraisal Note + full document package
Approve or reject with recorded reason in Credit Sanction Register
Authority matrix enforced: loans ≤ ₹5,00,000 require CFO + 1 Director; above ₹5,00,000 require CFO + 2 Directors — the UI must enforce this with countersignature workflow
Special-case flag: if borrower is a Director or relative, system routes for General Meeting approval
Approve invocation of SH-4 or undated cheque in default cases
5. Treasury Team (Chief Financial Controller + Senior Manager Finance)
Receive disbursement-ready files from Compliance
Create SAP Customer Code (or trigger API call to SAP)
Execute bank transfer via RBL Bank account
Chief Financial Controller countersigns as final authorised signatory
Post repayment entries in SAP on next working day
Record SAP journal entries for monthly interest accruals
Database schema (core entities)

Design these tables at minimum. All monetary fields in NUMERIC(15,2). All timestamps in UTC.

member_id, folio_number, member_type (individual | fpc), full_name, pan, aadhaar, address, email, phone, status (active | inactive), kyc_verified_at, kyc_due_at (= kyc_verified_at + 2 years)

LoanApplications
loan_application_id, loan_ref_number (LO00000001…), member_id (FK), nominee_name, nominee_aadhaar, nominee_pan, shares_pledged, land_area_acres, shareholding_limit (computed), land_limit (computed), eligible_limit (lower of two), requested_amount, approved_amount, purpose, loan_type (short_term ≤1yr | long_term), status (draft | submitted | under_assessment | approved | rejected | disbursed | closed), created_at, updated_at
AppraisalNotes
appraisal_id, loan_application_id (FK), prepared_by (FK users), eligibility_check (JSON — active member?, no default?, docs complete?, purpose valid?), risk_rating, income_evidence, recommended_amount, notes, submitted_at
SanctionDecisions
decision_id, loan_application_id (FK), decision (approved | rejected), reason, signatory_1 (CFO user_id), signatory_2 (Director 1 user_id), signatory_3 (Director 2 user_id — required if amount > 5L), signed_at_1, signed_at_2, signed_at_3, exception_reason
LoanDocuments
doc_id, loan_application_id (FK), doc_type (poa | tri_party | sh4 | cdsl_pledge | term_sheet | loan_agreement | bank_verification | checklist | appraisal_note | rejection_note | noc), file_url (S3), stamped (bool), notarised (bool), stamp_value, generated_at, signed_by (FK users)
Disbursements
disbursement_id, loan_application_id (FK), amount, bank_account, ifsc, transfer_ref (RBL UTR), sap_customer_code, disbursed_by (FK — Sr. Manager Finance), authorised_by (FK — Chief Financial Controller), disbursed_at
Repayments
repayment_id, loan_application_id (FK), amount, type (principal | interest), channel (direct | subsidiary), subsidiary_ref, received_at, sap_entry_at, posted_by (FK)
LoanLedger
Running balance per loan — principal outstanding, interest accrued, interest paid, total outstanding; updated on each repayment and on each year-end interest capitalisation (unpaid interest added to principal after 30 April)
DefaultEvents
event_id, loan_application_id (FK), event_type (grace_period_started | extension_granted | non_recoverable | sh4_invoked | cheque_invoked), reason, intentional (bool | null), actioned_by (FK), actioned_at
AuditLog
Every state change on every record — table_name, record_id, field_changed, old_value, new_value, changed_by (FK), changed_at. Immutable. Retained 8 years.
Core business logic rules never bypass
Loan limit calculation
shareholding_limit = shares_held × loan_value_per_share × 0.30 (currently ₹200/share × 30% = ₹60/share)
land_limit = land_area_acres × scale_of_finance (currently ₹20,000/acre)
eligible_limit = MIN(shareholding_limit, land_limit)
Both loan_value_per_share and scale_of_finance are configurable by admins (Board-approved changes only). Store in a SystemConfig table with effective dates and board resolution reference.
System must reject any disbursement where approved_amount > eligible_limit unless an exception is recorded in the exception register with CFO approval.
Approval routing
If approved_amount ≤ ₹5,00,000: require signatures from CFO + exactly 1 director
If approved_amount > ₹5,00,000 OR approved_amount > eligible_limit: require signatures from CFO + exactly 2 directors. Reason must be entered in exception register.
If borrower is a Director, or a relative of a Director: block normal approval flow, raise a "General Meeting Approval Required" task, link to Sec. 378ZK. Loan can only proceed after GM resolution is uploaded and marked confirmed by CS.
Document stamping gate
HARD BLOCK No disbursement can be triggered unless the checklist document has all four signatures: CS, Credit Manager, at least one Sanction Committee signatory, and Senior Manager Finance.
HARD BLOCK PoA and Loan Agreement must be marked stamped = true AND notarised = true before checklist can be finalised. Stamp value must be ₹500 for both.
SH-4 required if share_type = physical. CDSL pledge confirmation required if share_type = demat.
Repayment allocation
Partial repayments are always applied to principal first, then interest. No exceptions.
SAP repayment entry must be posted by Credit Manager no later than the next working day after payment is received and confirmed in bank statement.
If unpaid interest is not cleared by 30 April each year: system automatically adds it to principal and recalculates interest for the new FY on the revised principal. Borrower receives an email/SMS notification of this capitalisation event.
Interest rate is floating — tied to bank base rate. Any change must be communicated to borrowers via SMS/email before taking effect.
Default escalation engine
On missed scheduled repayment: system automatically creates a DefaultEvent with type grace_period_started, sets grace expiry = due_date + 90 days
At grace expiry if still unpaid: Credit Assessment Team must mark intent (intentional / non-intentional)
If non-intentional: system grants 1-year extension. Credit Manager must attach a note. New repayment date set.
If still unpaid after extension: loan flagged as non-recoverable. Credit Assessment Team prepares "Note for Non-Payment". Sanction Committee reviews and decides whether to invoke SH-4 or undated cheque. Both actions require Board approval, recorded in decision log.
KYC re-verification
System tracks kyc_due_at per member. 30 days before expiry: Credit Head receives task + borrower receives notification.
Expired KYC blocks any new loan application from that member.

Document generation specifications

All generated PDFs must be print-ready, bilingual-capable (English + Marathi for farmer-facing docs), and include the following on every page: SFPCL letterhead, loan reference number, date, member name, page number. Specific requirements per document type:

PoA: Stamp paper ₹500 value, notarisation placeholder, farmer + nominee signature blocks, witness block, CS acceptance block
Term Sheet: All 13 fields from SOP Section 4.9. Rate shown as floating with current effective rate. Signed by applicant and nominee. If amount > ₹5L, CFO + 2 Directors sign; if ≤ ₹5L, CFO + 1 Director signs.
Loan Agreement: Stamp paper ₹500 value, notarisation placeholder, borrower + witness signature blocks, full covenants, events of default, remedies
Checklist (Annexure H): Index of all documents in loan file, four signature blocks (CS / Credit Manager / Sanction Committee / Sr. Manager Finance)
NOC: Full repayment confirmation, loan reference, date, amount, declaration that SH-4 and blank cheque are returned, CS signature
Compliance and audit requirements
All records retained for 8 years — no hard deletes anywhere in the system. Use deleted_at soft delete pattern.
Audit log must be append-only. No update or delete endpoints on audit log table.
Section 186 Companies Act check: run quarterly; block new loan sanctions if total outstanding would breach 60% of paid-up capital + free reserves (or 100% of free reserves, whichever is higher) without board special resolution on file.
NBFC test: run quarterly; alert CFO if financial asset ratio or income ratio exceeds 45% (warning) or 50% (critical).
KYC: PAN and Aadhaar mandatory for every borrower and nominee. CKYC consent checkbox mandatory. Re-KYC cycle every 2 years enforced by system — expired KYC blocks new applications.
Maharashtra Stamp Act: system must record stamp duty paid, certificate number, and execution date for every stamped document. Disbursement is blocked if any required document has stamped = false.
Money-lending exemption: CS must confirm annually that SFPCL's Maharashtra money-lending exemption still applies. System surfaces an annual task for this; locks new applications if not confirmed.
Data access: role-based access at API level — borrowers can only query their own records. All KYC document URLs must be pre-signed with 1-hour expiry.
Key validation rules
Borrower must be active member (validated against member register) before application is accepted
Nominee cannot be a minor (age calculated from Aadhaar date of birth)
Witness on Loan Agreement and SH-4 must be an existing shareholder of SFPCL (validated against member register)
Loan purpose must be one of: crop production, agronomy inputs, post-harvest, allied agriculture activity — free text not permitted
Bank account details entered for disbursement must match the cancelled cheque uploaded — system shows both side-by-side for manual confirmation before initiating transfer
Signature consistency check: system flags if signature on PAN card, bank cheque, and KYC documents look inconsistent (manual review step, bank verification letter required)
No new loan can be disbursed to a borrower with an existing default on any loan from SFPCL or any subsidiary / associate company
Integration points
SAP: Customer code creation (Annexure I format — name, Aadhaar, PAN, address, email, loan ref), journal entry posting for disbursements, repayments, and interest accruals
CDSL: Pledge creation (Pledge Request Form fields: BO account, securities, PSN), pledge invocation (Invocation Request Form), pledge release (Unpledged Request Form)
RBL Bank: Outward RTGS/NEFT initiation, bank statement polling for incoming repayments, UTR capture
WhatsApp / SMS gateway: Borrower notifications in Marathi
Error states and exception handling
Every blocked action (disbursement blocked, approval blocked, duplicate application) must show a clear reason with the specific rule that is blocking it and the action required to unblock
Exception register is a first-class entity — any deviation from SOP (exceed limit, bypass step, partial KYC) must be documented here with justification, approver, and timestamp before the exception is permitted
All 10 top errors from the SOP must be enforced as hard UI blocks or warnings: non-member loan, missing PAN/Aadhaar, incomplete appraisal, missing witness signatures, disbursement before stamping, wrong bank details in SAP, missing year-end interest invoice, KYC not renewed, SH-4 invoked without board approval, NOC delay on closure
